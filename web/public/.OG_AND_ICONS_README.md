# Missing static assets

The following binary assets are referenced from `index.html` but are NOT generated
by code. They must be produced manually (Figma, Affinity, ImageMagick, etc.) and
dropped into `web/public/` before deploying to production.

## `og.png`

- Dimensions: **1200 × 630** (Open Graph / Twitter `summary_large_image`).
- Aesthetic: dark editorial card matching the site (background `#0a0a0a`, warm
  off-white text `#f5f5f0`).
- Typography: **Fraunces** serif, bold, large display size.
- Copy (single line or two lines):
  - Primary: `Roberto Rodríguez`
  - Secondary / kicker: `Portfolio conversacional`
- Optional: a small `RR` monogram top-left, a thin hairline divider, JetBrains
  Mono caption with the URL `robertorf.dev` bottom-right.
- Export as PNG, optimize (≤ 200 KB ideally).

## `apple-touch-icon.png`

- Dimensions: **180 × 180** PNG.
- Source: same monogram as `favicon.svg` (`RR` in Fraunces bold, `#f5f5f0`).
- Background: solid `#0a0a0a` (Apple touch icons should not be transparent —
  iOS will composite them on a white background otherwise).
- Export as PNG.

## How to verify after dropping in

1. `pnpm --filter web build` and check `dist/` contains both files.
2. After deploy, validate at:
   - https://www.opengraph.xyz/url/https%3A%2F%2Frobertorf.dev
   - https://cards-dev.twitter.com/validator
   - https://realfavicongenerator.net/favicon_checker
